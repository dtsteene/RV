# Software Used By The Model

The conceptual model above is implemented in the FEniCSx ecosystem {cite}`baratta2023dolfinx`. DOLFINx provides the parallel finite-element mesh, function-space, assembly, and solver infrastructure, while UFL represents the weak forms and constitutive expressions in symbolic Python notation before they are compiled by FFCx. This matters because the strain energy, stress, residual, and tangent can be written close to the mathematical form used in the previous sections.

The cardiac-specific pieces come from several libraries developed in the same ecosystem. `cardiac-geometries` {cite}`cardiac_geometries` generates and labels the biventricular mesh. The MPI-parallel fibre assignment is handled by `ldrb` {cite}`bayer2012novel`, which implements the Laplace-Dirichlet Rule-Based algorithm on DOLFINx meshes. `fenicsx-pulse` {cite}`fenicsx_pulse` provides the Holzapfel-Ogden material model, active-stress formulation, cavity-volume constraints, Robin supports, and prestressing machinery. The 0D circulation is provided by the `circulation` package {cite}`circulation`, following the closed-loop formulation of Regazzoni et al. {cite}`regazzoni2022cardiac`.

This thesis depends on field-level stress, strain, and work quantities rather than only on standard pressure-volume outputs. Several implementation details therefore matter: finite-element checkpoints are written with `adios4dolfinx`, activation is represented on region-aware simple-function spaces through `scifem`, and the final work-density postprocessing evaluates stress and strain at quadrature points. The implementation chapter explains how these choices were debugged and why the final checkpoint-based postprocessing path was retained.
